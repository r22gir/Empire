## LeadForge Block 1C — Campaign Draft/Review Send-Gating

## Date: 2026-04-11

### Bug Found

Campaign draft/review wiring had a safety mismatch:
- `send_all_reviewed()` only batches drafts with status `reviewed` or `edited`.
- `send_draft()` accepted any draft with `to_email`, including a newly generated `draft`.
- `LeadForgePageNew.tsx` showed the single-draft send button for any unsent draft with an email.

That made it possible for the review UI to attempt a real send before the draft had been reviewed or edited.

### Fix Applied

- Added a backend guard in `campaign_service.send_draft()` requiring draft status `edited` or `reviewed` before email service setup.
- Updated `DraftReviewPanel` so single-send and batch-send controls are only enabled for drafts that have an email and status `edited` or `reviewed`.
- Updated the single-send UI response handling to accept the backend's existing `{status: "sent"}` success shape.
- No provider calls, destructive data changes, or real outbound email sends were made.

### Verification

- Temp-DB LeadForge campaign smoke — ✅
  - created prospect, active campaign, manual email step, enrollment, due-step execution, draft creation, and draft edit
