# PHASE 3: THE DROP-SHIP ARBITRAGE PLAYBOOK
# ══════════════════════════════════════════════════════

## 3.1 How Drop-Ship Arbitrage Works (Step by Step)

Document the complete flow:

```
1. SCOUT: Find product cheaper on Platform A than selling price on Platform B
2. LIST: Create listing on Platform B at markup price
3. WAIT: Customer buys on Platform B
4. ORDER: Buy from Platform A, ship to Platform B customer's address
5. PROFIT: Keep the spread minus fees
6. REPEAT: Scale with more listings
```

**Key operational details:**
- How to handle tracking numbers (Platform A → Platform B customer)
- How to handle returns (customer returns to you, you return to Platform A?)
- How to handle price changes (Platform A raises price after you listed)
- How to handle out-of-stock (Platform A sells out)
- How to handle slow shipping (AliExpress 15-30 day shipping)
- Tax implications (sales tax nexus, 1099-K thresholds)

## 3.2 Account Health & Suspension Prevention

**THE BIGGEST RISK in arbitrage is account suspension.** Research:

```
- "Amazon seller account suspended arbitrage 2026"
- "eBay seller limits new account"
- "how to avoid eBay suspension drop shipping"
- "Amazon tracking number match policy drop ship"
```

Document:
- Platform policies on drop-shipping (what's allowed, what gets you banned)
- How to structure multiple seller accounts across platforms
- Tracking number requirements per platform
- Seller metrics you MUST maintain (shipping time, feedback, defect rate)
- How to gradually increase volume without triggering reviews

Save to: `~/empire-repo/docs/relist/account_health_playbook.md`

## 3.3 Automation Strategy with RelistApp

Map how RelistApp automates each step:

```
MANUAL (without RelistApp):
1. Browse Amazon for deals → 30 min
2. Check price on eBay → 10 min
3. Create eBay listing manually → 15 min
4. Wait for sale → passive
5. Buy on Amazon → 5 min
6. Enter tracking → 5 min
Total per item: ~65 min setup + ongoing

AUTOMATED (with RelistApp):
1. Product Scout scans Amazon deals → automatic
2. Profit Calculator checks spread on all platforms → automatic
3. Cross-Lister creates listings on eBay/Mercari/FB → automatic
4. Sale notification → instant
5. Auto-purchase from source → semi-auto (needs payment confirmation)
6. Tracking auto-forwarded → automatic
Total per item: ~5 min human time
```

**What RelistApp needs to do this at 1000/day scale:**
- Product discovery engine (scan Amazon BSR, clearance, deals)
- Real-time price comparison across 5+ platforms
- Bulk listing creation (title, description, images pulled from source)
- Order auto-routing (sale on Platform B → auto-order on Platform A)
- Price monitoring (alert if source price changes, auto-delist if margin disappears)
- Inventory sync (if source goes OOS, pause listing immediately)
- Analytics dashboard (profit per product, per platform, per day)

Research and document ALL risks:

**Account Risks:**

**Financial Risks:**

- Source out of stock after customer ordered
- Wrong item shipped
- Damaged in shipping
- Customer disputes/chargebacks
- Holiday shipping delays

- State sales tax requirements
