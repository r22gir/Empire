1. Executive Summary

Empire is an AI-powered autonomous business operating system designed to run day-to-day operations of service businesses with minimal human intervention. The platform combines 22 interconnected software products, an 18-desk AI workforce (MAX), and a multi-provider AI routing engine into a single deployable ecosystem.

The platform currently powers two active businesses — Empire Workroom (custom drapery & upholstery) and WoodCraft (woodwork & CNC) — while simultaneously developing a SaaS offering (ContractorForge) to bring the same capabilities to any service business.

2. Founder's Vision & Platform Philosophy

The Problem

Service businesses (contractors, workrooms, trades) juggle 8-12 separate software tools: QuickBooks for invoicing, a CRM, a scheduler, marketplace listings, social media, email marketing, inventory tracking, and more. Each tool costs $20-100/month, none talk to each other, and the owner spends hours daily on administration instead of their craft.

The Solution

Empire replaces the entire stack with a single platform where an AI assistant (MAX) handles operations autonomously. The founder focuses on the craft; MAX handles the business.

Design Principles

  • Founder Dogfooding First: Every feature is battle-tested on the founder's own businesses before SaaS release
  • Tool Blocks, Not Chat: MAX can only take action through verified tool calls — no hallucinated operations
  • Complexity-Driven AI Routing: Simple queries use cheap models ($0/call); complex analysis routes to premium AI — optimizing cost without sacrificing quality
  • Desk Separation: Each business function has its own AI agent with scoped tools, personality, and model preference
  • Complete Audit Trail: Every AI call, every tool execution, every cost — logged and queryable
  • Self-Hosted First: Runs on a single server (Intel Xeon, 32GB RAM) with Cloudflare tunnel for external access — no cloud dependency

Origin Story

Empire was inspired by the concept of local AI agents running on commodity hardware. Starting on a Beelink EQR5 mini-PC, the founder built a proof-of-concept connecting OpenClaw + Ollama + Claude API. Within weeks, it evolved into a full business operating system. The platform migrated to a Dell PowerEdge workstation (EmpireDell) as scope expanded, and now comprises 223,000+ files across 22 products.

5. MAX — AI Orchestration Engine

MAX is the core intelligence layer of the Empire platform. It is not a chatbot — it is an AI workforce consisting of 18 specialized agents (desks), each with scoped responsibilities, tools, and AI model preferences.

Key Architecture Decision: MAX can only execute actions through structured tool blocks — never through free-form text generation. This eliminates the hallucination problem that plagues other AI assistants. Every quote, invoice, email, and code change is executed via verified tool calls with real database operations.

5.3 AI Model Routing

MAX uses a complexity-based routing system that automatically selects the most cost-effective AI model for each interaction:

ComplexityExamplesModel Chain
SimpleGreetings, yes/noGemini Flash → Grok → Groq
ModerateConversations, listsGrok → Groq → Claude Sonnet
ComplexAnalysis, strategyClaude Sonnet → Grok → GPT-4o
CriticalCode, deploymentClaude Opus → Claude Sonnet

Fallback guarantee: If any provider fails, the system automatically cascades to the next provider within 15-30 seconds. Local models (OpenClaw, Ollama) serve as the final fallback — ensuring MAX never goes offline.

5.4 Autonomous Operations

MAX runs scheduled tasks without human intervention:

  • 8:00 AM — Morning briefing: system stats, task counts, alerts
  • 9:00 AM — Overdue task check with notifications
  • Monday 10:00 AM — Weekly sales follow-up generation
  • Friday 5:00 PM — Weekly business report
  • 11:00 PM — Nightly brain sync (persist conversation context)

5.5 Persistent Memory

MAX maintains 3,041 persistent memories across conversations, enabling context continuity. The memory system stores customer preferences, business decisions, conversation summaries, and operational learnings. Memory is loaded dynamically with each interaction (4KB limit) and summarized nightly.

7.3 Safety Guardrails

  • Prompt Injection Detection: 14 pattern signatures blocked (jailbreak, DAN mode, ignore previous, etc.)
  • Blocked Topics: Malware, explosives, hacking instructions automatically rejected
  • Output Sanitization: API keys (sk-*, xai-*) stripped from all responses
  • Hallucination Detection: Fabrication phrase markers flagged in output
  • File Safety: Truncation protection (blocks writes reducing file size >50%), auto-backup before overwrites, critical file guard on core system files
  • Shell Safety: Allowlisted commands only; rm -rf, pkill -f, kill -9, dd permanently blocked
  • Path Validation: All file operations restricted to ~/empire-repo/

10. Challenges, Risks & Mitigations

10.1 Technical Challenges Encountered

AI Model Reliability

Challenge: Single-provider AI dependency risks service outages when API providers experience downtime or rate limiting.

Solution: Built a 5-tier fallback chain (Grok → Claude → Groq → OpenClaw → Ollama). Local models ensure MAX never goes fully offline. Each provider has configurable timeouts (10-30 seconds) before escalation.

Cost Control

Challenge: AI API costs can spiral quickly with Claude Opus at $75/M output tokens.

Solution: Complexity-based routing ensures 80%+ of interactions use the cheapest appropriate model. Budget alerts at 80% with auto-switch to local models at 95%. Per-business cost tracking prevents cross-subsidization.

Tool Execution Safety

Challenge: Giving an AI access to shell commands, file operations, and database queries introduces significant risk.

Solution: 3-tier access control (Auto/Confirm/PIN), command allowlisting, file truncation protection, path validation, and complete audit logging. Every tool execution is recorded with user, timestamp, action, and result.

Hardware Limitations

Challenge: Running 10+ services on a single server with an unstable GPU driver.

Solution: Efficient SQLite (WAL mode) databases, lightweight Next.js builds, Cloudflare Tunnel for CDN/DDoS protection. GPU driver fix is planned (sudo ubuntu-drivers autoinstall) but non-blocking for current operations.

Context Window Management

Challenge: AI conversations lose context over long sessions, leading to repeated mistakes and forgotten instructions.

Solution: Persistent memory system (3,041 entries), nightly brain sync, conversation summaries (99 stored), and dynamic system prompt loading with ecosystem catalog awareness.
